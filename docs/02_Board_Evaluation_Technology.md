# 中国象棋计算机博弈局面评估技术研究

## 基本信息
- **标题**: Research on Technology of Board Evaluation in Chinese Chess Computer Game
- **作者**: 黄鸿, 林健, 任雪梅
- **来源**: 北京理工大学, 中文期刊
- **关键词**: 中国象棋, 计算机博弈, 局面评估

## 摘要

在以博弈树搜索为核心的人机对弈系统中，局面评估是决定程序智能水平的关键。本文基于对相对成熟的中国象棋软件的研究，将各类局面评估方法分为四类：静态单子型、未来局势型、象棋知识型及局面附加信息。阐述各类方法的原理、实现策略、优化方法及其与相关博弈模块的关系。

## 主要内容

### 1. 局面评估方法分类

#### 1.1 静态单子型评估

**原理**: 对棋盘上每一个棋子考虑其种类和位置，依重要性与位置优劣决定评估值，然后累加。

| 棋子 | 将 | 士 | 象 | 车 | 马 | 炮 | 兵 |
|------|----|----|----|----|----|----|-----|
| 价值 | 10000 | 110 | 110 | 300 | 600 | 300 | 70 |

**子公式**:
```
Strength = Σ(wi × Value_i)  
其中 wi 为棋子重要性权重，因局面变化而改变
```

**位置价值**: 同一棋子在不同位置价值不同
- 例: 兵过河价值大幅提升，与敌方将/帅距离越近价值越高

#### 1.2 未来局势型评估

考虑局面产生是否利于我方未来发展。

**棋子灵活度**:
```
E_mobility = Σ(c_kind × n_move)
```
- c_kind: 着法价值（通常与子力价值正相关但更小）
- n_move: 可行着法数

**棋盘控制力**:
- 我方可控制的点：己方棋子可到达的位置
- 敌方可控制的点：敌方棋子可到达的位置
- 威胁与保护: 评估双方在关键位置的控制力对比

| 考查点 | 我方着子A可到达的且B受保护 | 目标点是敌方着子B |
|--------|---------------------------|-------------------|
| 结果   | B被A攻击，加分             | B被A攻击，加分    |
|        | B被A保护，减分             | A被反防，减分     |

#### 1.3 象棋知识型评估

利用中国象棋丰富的知识体系。

**棋子配合打法**:
- "连环马"、"马后炮"、"车沉底炮碾丹砂"等经典战术
- 需要判断绝对位置（要害位置才能充分发挥威力）
- 配合打法只有在棋盘要害位置才能充分发挥威力

**定"形"与定"势"**:
- **定"形"**: 子力位置 (如：特殊位置的单马双象可守单车)
- **定"势"**: 子力配置 (如：单马必胜单士)
- 实现方式：直接统计棋子数目或通过残局库识别

#### 1.4 局面附加信息

与整个对局过程相关的其他因素。

**先行权分值**:
```
- 开局时先行权分值大
- 随局势演化而减小
- 残局可能出现"无等着"的负分值
```

**特殊局势常值**:
- 必胜、必败局面
- 循环局面
- 自然限招局面
- 时限逼近

### 2. 局面评估与相关模块

#### 2.1 博弈树搜索

```
E_return = E_sum,us - E_sum,opponent
```

- 正值：对我方有利
- 负值：对敌方有利
- 局面评估系统与搜索引擎的耦合点

**评估与搜索的平衡**:
```
性能 = 速度 × 知识

图示关系: 知识越丰富，需要的速度越高
        速度快，知识相对简单
```

#### 2.2 着法生成

着法生成与局面评估的协同优化：
- 棋盘扫描、模板匹配、预置表等方法
- 定"势"和灵活度评估所需数据可在着法生成时直接获取
- 控制力和配合打法评估：对生成该着法的棋子运行"类着法生成"

#### 2.3 知识库

| 比较项目 | 象棋知识型评估 | 知识库 |
|---------|----------------|--------|
| 考查对象 | 局面几个棋子关系关位置 | 整个棋谱、所有棋子位置 |
| 实现方式 | 融入局面评估系统 | 由数据库查询 |
| 查询性 | 可应对全局局面 | 针对特定局面 |
| 执行效率 | 需要计时、速度快 | 速度快 |
| 应用范围 | 兼不大适用场外知识 | 专门为某预知识库 |

### 3. 局面评估的优化

#### 3.1 参数初值选取

**经验定值原则**:
- 子力价值：车 > 马，马 > 兵
- 规格化原则：先确定不同参数的比例系数，再乘以倍数
- 约束原则：参数应符合先验约束条件

#### 3.2 优化算法

**(1) 爬山法**
- 每次对一个参数进行小幅修改
- 测试棋力是否提高
- 缺点：只能找到局部最优解

**(2) 模拟退火**
- 基于爬山法，引入蒙特卡罗采样
- 用Metropolis重要性采样概率进行随机跳跃
- 逐步降低接受差解的概率
- 缺点：速度较慢，但结果更优

**(3) 遗传算法**
- 模拟生物染色体选择、遗传、变异机制
- 维护多组较好参数，通过交叉和选择迭代
- 关键：适应度函数定义
- 适应度计算：与高手同局面对弈的着法比照，或与其他程序对局胜率

**(4) 神经网络**
- 参数权重作为神经元
- 大量棋谱为输入层，最终评估值为输出层
- 用适应度函数给予奖励和惩罚
- 优势：避免片面的统计角度，更具人性化
- 可独立运行作为具备自学习功能的评估系统

### 4. 核心观点

#### 评估函数的关键特性
1. **基于象棋知识**: 需要大量象棋知识作为专家系统
2. **多层次参数**: 公式(1)中的权重ki和子评估函数的各参数都是待优化对象
3. **自学习能力**: 优化算法使参数体系具备一定的自学习能力
4. **快速迭代**: 在较短时间和较少人工干预下趋近最优解

#### 与其他模块的关系
1. **与搜索的权衡**: 评估复杂度与搜索深度的平衡（见图2）
2. **与着法生成的融合**: 充分利用着法生成过程的信息
3. **与知识库的补充**: 评估函数处理动态/一般情况，知识库处理特殊情况

## 关键数据结构

### 评估函数基本形式
```
E_sum,player = Σ(ki × Ei(p))  ... (1)
Ei(p) = Ei(x1, x2, ..., xk)   ... (2)
```

其中：
- E_sum,player: 局面最终评估值
- Ei(p): 第i种子评估函数
- ki: 权重系数
- xj: 局面局部信息
