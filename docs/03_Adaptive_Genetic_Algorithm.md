# 中国象棋计算机博弈系统评估函数的自适应遗传算法实现

## 基本信息
- **标题**: Implementation of Adaptive Genetic Algorithm of Evaluation Function in Chinese Chess Computer Game System
- **作者**: 王骄, 王涛, 罗燕红, 许新和
- **来源**: Journal of Northeastern University (Natural Science), Vol. 26, No. 10 (2005), pp.949-952
- **关键词**: 中国象棋, 计算机博弈, 博弈树, 评估函数, 锦标赛算法, 自适应遗传算法

## 摘要

自适应遗传算法(AGA)用于解决中国象棋计算机博弈的参数优化问题。通过博弈树搜索和评估函数优化，实现参数的自动调整。基于该方法设计了一个离线自学习系统。实验结果表明，AGA提高了所开发程序的性能。

## 核心贡献

### 1. 遗传算法在象棋中的应用

#### 1.1 基本框架

**种群结构**:
- 种群大小：通常20-100个个体
- 每个个体：一组象棋评估参数
- 适应度函数：参与自对弈竞技的胜率

**适应度定义**:
```
fitness = (赢的局数) / (总对弈局数)
```

#### 1.2 遗传算子

**(1) 选择机制 - 改进的锦标赛选择**

标准方法缺点：速度过慢

**改进方案**:
```
1. 将包含N个个体的种群随机分成M组
2. 在组内进行锦标赛训练
3. 每个组的冠军为筛选出的最佳
4. 对M个冠军做交叉和变异产生100个新个体
5. 填满并形成新种群
```

**效果**: 相比标准方法节省约50%时间

**(2) 均匀交叉**

交叉点选择：参数间隔点作为潜在交叉点

```python
# 基于交叉率Pc生成掩码
mask = 生成与参数个数等长的二进制串
# 掩码中的片段表示哪个父个体提供变量
子个体 = 通过掩码和父个体确定
```

**(3) 变异操作**

基于变异率Pm进行基因翻转：
```python
if random() < Pm:
    gene[i] = flip(gene[i])  # 二进制翻转
```

**目的**: 保持种群多样性，防止非成熟收敛

### 2. 自适应遗传算法(AGA)

#### 2.1 动态参数调整的必要性

传统GA中Pc和Pm是固定的：
- **Pc过大**: 新个体产生快，但高适应度个体结构易破坏
- **Pc过小**: 搜索缓慢甚至停滞
- **Pm过小**: 不易产生新结构
- **Pm过大**: 变成纯随机搜索

**问题**: 目前无通用方法一次性确定最优Pc和Pm，需通过实验反复调试

#### 2.2 自适应策略

```
Pc(t) = {
    pc1 - (pc1 - pc2) * (f' - f_avg) / (f_max - f_avg),  if f' >= f_avg
    pc1,                                                    if f' < f_avg
}

Pm(t) = {
    pm1 - (pm1 - pm2) * (f_max - f') / (f_max - f_avg),  if f' >= f_max
    pm1,                                                    if f' < f_max
}
```

其中：
- pc1 = 0.9, pc2 = 0.6, pm1 = 0.1, pm2 = 0.001
- f_max: 群体最大适应度
- f_avg: 群体平均适应度
- f': 待交叉(或变异)个体的适应度

**优势**:
- Pc和Pm随适应度自动改变
- 在保持群体多样性的同时，保证收敛性
- 相对某个解提供最佳Pc和Pm

## 实现建议

1. **初始化**: 基于经验知识初始化参数，如子力价值
2. **群体大小**: 20-30个个体较平衡
3. **迭代次数**: 至少300-600代看到明显效果
4. **自对弈时间**: 深度搜索(如4-5层)才能有效评估
5. **收敛判定**: 连续50代无进步则停止
